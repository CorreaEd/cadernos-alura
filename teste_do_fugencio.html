Fugêncio aprendeu que a estrutura mínima para criar seus programas é:

<meta charset="UTF-8">
<script>
</script>

Confiante do seu entendimento, ele criou o arquivo teste_do_fugencio.html e escreveu o seguinte código:

<meta charset="UTF-8">

alert("Este é um popup feito com JavaScript");
<script>
    <h1>Primeiro programa</h1>
</script>

Depois de abrir em seu navegador ele verificou duas coisas:

1 - O popup da instrução alert não foi exibido, além disso a instrução foi indevidamente exibida na página.

2 - Para entender o que houve de errado, ele abriu o depurador (debugger) que todo navegador tem, foi na aba console e viu que há um erro na linha 5.

O que deveria ser alterado para que o código de Fugêncio funcionasse?

Fugêncio confundiu-se, ele escreveu código HTML no mundo JavaScript e código JavaScript no mundo HTML. Mas de que mundo estamos falando mesmo? Quando falamos do mundo do JavaScript, nos referimos a tudo que esta entre as tags <script>...</script>.

Quando o navegador encontra a TAG <script>..</script> ele passa a interpretar todo aquele código como JavaScript, é como se ele trocasse de óculos temporariamente e esses óculos lhe fornecessem um entendimento especial daquele código. Tudo que está fora da TAG script é considerado código HTML e o navegador irá analisar este código como tal. Não existe no mundo HTML a instrução alert e nem a TAG <h1> no mundo JavaScript, contudo, podemos corrigir o código de Fugêncio mudando o código de lugar.

O código final fica assim:

<meta charset="UTF-8">

<h1>Primeiro programa</h1>
<script>
    alert("Este é um popup feito com JavaScript");

</script>

Agora temos cada instrução no seu devido lugar!

Primeiro programa
